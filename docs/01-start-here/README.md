# Start here

> _What this page covers:_ The newcomer journey — what to read, in which order.
> _Who it's for:_ Anyone who just got access to this repo.

If you're new to Overbookd, read these pages **in order**. The whole journey is ~10 pages and gets you from "I just cloned the repo" to "I shipped my first MR".

| # | Page | What you'll learn |
|---|---|---|
| 1 | [Prerequisites](./01-prerequisites.md) | Tools, accounts, and access you need before anything else |
| 2 | [Local setup](./02-local-setup.md) | Boot the app on your machine using Docker + Traefik |
| 3 | [Repo tour](./03-repo-tour.md) | A 5-minute walk through the layered monorepo |
| 4 | [Your first feature](./04-first-feature.md) | End-to-end walkthrough: domain → API → UI → MR |
| 5 | [Troubleshooting](./05-troubleshooting.md) | The setup errors people actually hit, with fixes |

When you finish the journey, jump into the rest of [`docs/`](../README.md):

- [`02-architecture/`](../02-architecture/README.md) — how the code is laid out and why
- [`03-business/`](../03-business/README.md) — festival concepts, glossary, per-domain explanations
- [`04-conventions/`](../04-conventions/README.md) — commits, testing, recipes
- [`05-operations/`](../05-operations/README.md) — local dev gotchas + deployment topology
- [`06-reference/`](../06-reference/README.md) — scripts, env vars, tooling
